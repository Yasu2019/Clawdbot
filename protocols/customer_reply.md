# Customer Reply

## Purpose
- 顧客、仕入先、社内向けの返信文を、丁寧かつ過剰に責任を認めず作成する。

## Procedure
1. 結論を先に書く。
2. 事実と今後の対応を分けて書く。
3. 未確認事項は断定せず、確認中と明示する。
4. 添付資料がある場合は役割を明記する。
5. お詫び、依頼、確認のトーンを使い分ける。

## Tone Rules
- 丁寧で控えめな日本語を使う。
- 断定しすぎない。
- 不要に責任を認めすぎない。
- 相手に次の行動がある場合は明確にする。

## Output Template
```markdown
# Customer Reply Draft

## Recipient Type
- 顧客 / 仕入先 / 社内

## Intent
- お詫び / 依頼 / 確認 / 報告

## Facts
- 

## Draft
件名:

本文:
いつもお世話になっております。

結論:

事実:

今後の対応:

添付:

以上、よろしくお願いいたします。
```

## Rules
- 顧客向けは、結論、現状、次回報告予定の順で書く。
- 社内向けは、依頼事項と期限を明記する。
- メール起点なら `mail_todo_extract` で返信要否を先に確認する。
