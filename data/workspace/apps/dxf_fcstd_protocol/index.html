Current overlap

既存アプリ apps/dxf2step はすでに FreeCAD ベースで DXF から 3D モデルを生成しています。 Worker 側にも FreeCADCmd、レイヤー分割、厚み推定、STEP 出力の流れがあります。

  • 既存 UI: DXF 読み込み、レイヤー確認、ジョブ実行、プレビュー
  • 既存 backend: /api/dxf2step
  • 既存 FreeCAD pipeline: headless 実行済み

What the protocol adds

protocol pack は、単なるメッシュ生成ではなく、Feature tree を意識した FCStd 自動生成へ寄せる考え方です。 目標は「Body / Sketch / Pad / Pocket / Hole の構造化」と、report.json などの説明可能な出力です。

  • 主出力を FCStd に寄せる
  • Feature 名と生成ログを残す
  • 失敗時に report と preview を返す

Recommended next step

まずは既存 dxf2step を改修して、出力選択に FCStd を追加するのが最短です。 新規サービスを増やすより、既存ジョブ管理と FreeCAD 実行基盤をそのまま使えます。

Suggested path

DXF -> layer split -> FreeCAD feature build -> FCStd + STEP/STL + report
