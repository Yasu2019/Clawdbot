# UNITY AUTONOMY PROTOCOL (UAP-2026)

> **Purpose:** Enable Clawdbot to autonomously generate and verify Unity scenes/simulations.
> **Mechanism:** C# Script Injection -> Host BatchMode Execution -> Log Analysis.

## 1. Roles

- **Brain (Antigravity):** Writes C# Scripts (`AutoManager.cs`) and Scene files (`.unity`).
- **Brawn (Clawdbot Host):** Executes Unity in Headless BatchMode.
- **Feedback:** `unity_editor.log` scrutiny.

## 2. Environment (Verified)

- **Executable:** `C:\Program Files\Unity\Hub\Editor\6000.3.6f1\Editor\Unity.exe`
- **Project Path:** `D:\Clawdbot_Docker_20260125\data\workspace\projects\unity_viz`
- **Log Path:** `D:\Clawdbot_Docker_20260125\data\workspace\projects\unity_viz\Logs\editor.log`

## 3. Operational Loop

1. **Generate:** Antigravity writes a C# script (e.g., `Assets/Scripts/SimulationRunner.cs`) that performs a task and calls `EditorApplication.Exit(0)`.
2. **Execute:** Clawdbot runs the specific "Execute Method" command on Host.

    ```powershell
    & "C:\Program Files\Unity\Hub\Editor\6000.3.6f1\Editor\Unity.exe" -batchmode -nographics -projectPath "D:\Clawdbot_Docker_20260125\data\workspace\projects\unity_viz" -executeMethod SimulationRunner.Run -logFile "Logs\editor.log"
    ```

    > **⚠️ IMPORTANT:** This command MUST be executed on the **Windows Host (PowerShell)**.
    > The Docker Container **CANNOT** run this directly.
    > If you are the Docker Agent: Write the C# script, then **Request Execution** via `notify_user` or ask Antigravity.

3. **Verify:**
    - Clawdbot checks `Logs\editor.log` for "Exception" or "Error".
    - Clawdbot checks for output artifacts (Screenshots / CSVs generated by the script).

## 4. Safety Rules

- **No Graphics:** Always use `-nographics` for automation unless taking screenshots (in which case, ensure screen is unlocked or use software renderer).
- **Explicit Exit:** Scripts MUST call `EditorApplication.Exit(code)` to prevent the process from hanging forever.
